; *********************************************************************
; Core options for the game.
; Obviously, we recommend you play with the default values since those have been tailored for what we think is the best experience,
; but if something is bugging you, most game parameters can be changed.
; We recommend you make the changes to the GameOptions.txt file that is generated for you in /My Documents/Fell Seal/ rather than in
; this file, since this file will be updated by a new build, which would destroy your changes.
; We also recommended only updating the /My Documents/Fell Seal/ version of this document with values you are actually changing,
; to prevent stale values from persisting over our own updates.
;
; The way the game loads these files is it loads the game's builtin GameOptions.txt, then the /My Documents/Fell Seal/customdata/GameOptions.txt file is loaded on top.
;
; ***** NOTE: ****
; Do NOT copy all the values from this file into your customdata/GameOptions.txt file!
; You should only add/copy headers and entries that you're adding or modifying.
; Copying all these values means that whenever the base values of the game change during an update,
; your mod will keep overriding them (even for values your mod isn't actually using or caring about).
; This will prevent balance updates from applying correctly to the game if you're using a mod.
; *********************************************************************

; *************** Loot Drops entries ******************************
; Each table contains an 'empty' value, which is the chance of getting 'nothing'
; then each drops are listed 1 by 1, with the format:
; dropX=itemName, chanceOfDropping
; eg; drop0=ING-00, 25
; all chances for drops are tallied together then scaled to fit the space of (100-empty)
; eg: empty=65 (65% chance of getting nothing), that leaves 35% of getting 'something'.
; drop0=ing-00, 25
; drop1=ing-01, 50
; this will give final odds -> nothing: 65%, drop0: 35% * 25/(25+50), drop1: 35% * 50/(25+50)
;
; fixeddrop0=ing-00, 1
; same as above, but 'fixeddrop' means the % chance is fixed and will not be normalized as the above. Mainly used for special drops to make sure they don't get normalized to a high value by mistake.
;
; [LootTableY-X] the X is the story index, the Y is the number of the loot table (you can assign a specific tabzzle to maps).
; This table will be used when the story has reached that index. The highest storyindex possible table is the one used.
;
; Each encounter sets its lootable in encounters.txt and enemies can have custom loot info as well.


; Default values
[LootTableDefault]

; These values are used for every LootTable by default. The loot table can override them.
